Laravel CRUD

    CRUD adalah singkatan dari Create, Read, Update, dan Delete. Ini adalah operasi dasar yang digunakan untuk mengelola data di aplikasi web, dan Laravel menyediakan cara yang sangat efisien untuk menangani CRUD melalui Eloquent ORM (Object-Relational Mapping). Berikut adalah gambaran umum tentang masing-masing operasi CRUD:
  • Create (Membuat Data): Menambahkan data baru ke database. Di Laravel, ini dilakukan menggunakan metode seperti create() atau save() pada model.
      - Create
  • Read (Membaca Data): Mengambil data dari database. Ini bisa berupa semua data atau data spesifik berdasarkan kondisi tertentu. Metode seperti all(), find(), findOrFail(), atau where() digunakan untuk membaca data.
      - Read
      - Read with id
      - Read: Conversion date time
  • Update (Memperbarui Data): Memperbarui data yang sudah ada di database. Ini dilakukan dengan metode seperti update() atau dengan mengubah properti model dan kemudian memanggil save().
      - Update
  • Delete (Menghapus Data): Menghapus data dari database menggunakan metode delete() atau destroy() pada model.

    Anda perlu menyiapkan database di MySQL. Sebagai contoh saya menggunakan db_perpustakaan dengn tabel book.

    2. Tambahkan file Book.php pada models

    Buat file dengan nama Book.php pada app/Models/Book.php pada laravel anda. 

                    
                
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class Book extends Model
{
    use HasFactory;
    protected $primaryKey = 'Book_ID';
    protected $fillable = [
        'Book_ID',
        'Title',
        'Author',
        'Publisher',
        'Year_Published',
        'Genre',
        'ISBN',
        'Copies_Available',
        'Pict'
    ];
    public $incrementing = false;
    protected $keyType = 'string';

    public $timestamps = false;
}
  • use HasFactory;: Menggunakan trait HasFactory untuk memungkinkan pembuatan instance model menggunakan factory.
  • protected $primaryKey = 'Book_ID';: Mengatur primary key tabel menjadi Book_ID alih-alih default id.
  • $fillable: Menentukan atribut yang dapat diisi secara massal melalui metode seperti create() dan update().
  • $incrementing class="px-6 dark:text-gray-100 mb-2">$incrementing = false;: Menandakan bahwa primary key tidak auto-increment.
  • $keyType = 'string';: Menentukan tipe primary key sebagai string.
  • $timestamps = false;: Menonaktifkan timestamps karena tabel tidak memiliki kolom created_at dan updated_at.

    Read: Conversion date time

    Untuk mengonversi tanggal dari format 2024-08-01 menjadi 1 Agustus 2024 di Laravel, Anda bisa menggunakan Carbon, yang merupakan library untuk manipulasi tanggal dan waktu yang sudah terintegrasi dengan Laravel.

    1. Format Tanggal di Blade

    Sebagai contoh anda menggunakan kolom Membership_Date untuk menyimpan tanggal. Jika Membership_Date sudah merupakan objek tanggal atau string tanggal yang valid, Anda dapat memformatnya langsung seperti ini: 

                    
                
@php
    use Carbon\Carbon;

    $formattedDate = Carbon::parse($members->Membership_Date)->translatedFormat('j F Y');
@endphp

{{ $formattedDate }}

    Disini kita membuat variabel formattedDate yang berisi konversi tangggal dari Membership_Date.

    2. Set Locale untuk Bahasa Indonesia

    Untuk memastikan bulan ditampilkan dalam bahasa Indonesia, pastikan Locale diatur ke id. Anda bisa menambahkan ini di awal, misalnya di file AppServiceProvider pada direktori app/Providers/ 

                    
                
use Carbon\Carbon;

public function boot()
{
    Carbon::setLocale('id');
}

    3. Jika Membership_Date Adalah Null

    Pastikan untuk menangani kasus di mana Membership_Date bisa saja null, agar tidak ada error 

                    
                
@php
    $formattedDate = $members->Membership_Date 
        ? Carbon::parse($members->Membership_Date)->translatedFormat('j F Y') 
        : 'Tanggal tidak tersedia';
@endphp

    Update (Memperbarui Data)

    1. Menambahkan file di Controller

    Hampir sama seperti create data, Dalam Controller kita tambahkan method edit untuk menampilkan view dari form tambah data. Dan update untuk menyimpan data yang diubah. 

                    
                
  public function edit($id)
   {
      $books = Book::where('Book_ID', $id)->firstOrFail();
      return view('books/booksupdate', compact('books'));
   }

  public function update(Request $request, $id)
  {
      $books = Book::where('Book_ID', $id)->firstOrFail();

      $request->validate([
          'Title' => 'required|string',
          'Author' => 'required|string',
          'Publisher' => 'required|string',
          'Year_Published' => 'required|integer',
          'Genre' => 'required|string',
          'ISBN' => 'required|string',
          'Copies_Available' => 'required|integer',
      ]);


      // Update other fields
      $books->Title = $request->Title;
      $books->Author = $request->Author;
      $books->Publisher = $request->Publisher;
      $books->Year_Published = $request->Year_Published;
      $books->Genre = $request->Genre;
      $books->ISBN = $request->ISBN;
      $books->Copies_Available = $request->Copies_Available;
      
      $books->save();

      return redirect()->route('books.detail', ['id' => $books->Book_ID])->with('success', 'Data buku berhasil diperbarui.');
  }

    Disini kita menggunakan request, jika pada create hanya menggunakan validate pada update menggunakan request. Dan juga pada books memvalidasi jika id adalah Book_ID.

    2. Mengatur pada route web.php

    Dalam route kita atur menjadi seperti ini. 

                    
                
// Route untuk halaman update buku
Route::get('/books/{id}/edit', [BookController::class, 'edit'])->name('books.edit');
Route::put('/books/{id}', [BookController::class, 'update'])->name('books.update');

    Disini kita membuat dua route untuk update data. Yang pertama digunakan untuk tampilan dan yang kedua digunakan untuk jalur data. Jadi disini menggunakan get dan put. Disini menggunakan id karena kita akan mengubah hanya satu data secara uniq maka menggunakan id.
